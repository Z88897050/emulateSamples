通常的做法是先对语言中所有字符组成的字符集按照一定的规则进行编码。最直接的编码方式是为每个字符在给定的范围之内分配一个整数序号。这个整数序号被称
为该字符的代码点。

对于一个字符集来说，其中每个字符的代码点的分配方式是固定的，一般由相关的标准化组织来制定。

在经过这种编码之后，之前的字符集就变成编码字符集。如：ASCII,Unicode,GBK

有了编码字符集之后，下一步要做的就是把字符的代码点映射到计算机中。

最直接的映射方式就是一一映射，即把字符的代码点直接映射到对应的数字。如ASCII中的字母"A"就用整数65表示。但是并没有考虑字符集中的字符总数和字符的
使用频率。

从节省空间和提高使用率的角度出发，对于Unicode这样的编码字符集，更好的办法是采用多个代码单元来表示一个字符。

一个代码单元是编码时使用的最小单元。

Unicode一共定义了1,114,112个代码点，值的范围从0到Ox10FFFF，代码点被分成17个区域，每个区域为一个平面，每个平面中包含65536个字符。

UTF-8是一种变长的编码格式，其中每个代码单元是8位。一个Unicode字符的代码点会被映射到1到6个代码单元，这种变长编码格式的一个好处是可以减少所需要
的存储空间，对于Unicode中的常见字符，仅用一个代码单元就可以表示，另外，UTF-8编码的前128个字符与ASCII编码完全一致。

UTF-16的每个代码单元是16位，每个Unicode代码点会被映射到1或2个16位的代码单元上。

就utf-8和utf-16的使用场景而言，前者更多是作为字符传输时的编码格式，后者更多是作为字符在系统内部的表示方式。
